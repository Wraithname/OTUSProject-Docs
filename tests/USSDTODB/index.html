    13. Cервис получения данных по процессу миграции объектов БД¶

    13.1 Тест-кейсы MVP¶

    13.1.1 USSDTODB-ACUS01¶

    13.1.1.1 Основной сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Тестировщик видит главную страницу сервиса миграции объектов
    2 Убедиться в наличии привязки объектов БД к проектам и модулям Тестировщик видит что объекты привязаны к проектам и модулям
    3 Убедиться в наличии навигационной панели по проектам и модулям Выполняется переход в личный кабинет клиента
    4 Нажать на кнопку запуска обновления структуры проектов После небольшой задержки (до 2-х секунд) структура проектов обновлена
    5 Убедиться в наличии выпадающего списка для навигации Выпадающий список присутствует на форме
    6 Выбрать в выпадающем списке пункт "По проекту" Список видоизменяется на навигацию по проекту
    7 Выбрать в выпадающем списке пункт "По модулю" Список видоизменяется на навигацию по модулю
    8 Выбрать в выпадающем списке пункт "По проекту и модулю" Список видоизменяется на навигацию по проекту и модулю

    13.1.1.2 Негативный сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Тестировщик видит главную страницу сервиса миграции объектов
    2 Нажать на кнопку запуска обновления структуры проектов После 2-х секунд ожидания появляется сообщение "Возникла непредвиденная ошибка"

    13.1.2 USSDTODB-ACUS02¶

    13.1.2.1 Основной сценарий через управление мышью¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Тестировщик видит главную страницу сервиса миграции объектов
    2 Нажать на кнопку открытия фильтра Открыто окно фильтрации
    3 Убедиться в наличии текстовых полей для фильтрации у атрибутов сущности Поля для фильтрации присутствуют
    4 Убедиться в наличии выпадающих списков для полей с ограниченным набором допустимых значений Выпадающие списки в наличии и содержат фиксированные значения
    5 Ввести "like %Банки%" в поле "Проект" и нажать на "Ок" Окно фильтрации закрыто, объекты на главной странице отфильтрованы по проекту "Банки"
    6 Нажать на горячую клавишу открытия фильтра (F7) Открыто окно фильтрации
    7 Нажать на "F8" Окно фильтрации закрыто, Пустой фильтр применен

    13.1.3 USSDTODB-ACUS03¶

    13.1.3.1 Основной сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Тестировщик видит главную страницу сервиса миграции объектов
    2 Открыть фильтр и задать любые условия фильтрации (например, в поле "Проект" указать like `%Банки%") Условия фильтрации заданы и отображаются в форме фильтра
    3 Нажать на кнопку сохранения фильтра на панели инструментов Появляется диалоговое окно для ввода названия фильтра
    4 Ввести название фильтра (любое) и нажать "Ок" Фильтр сохранён, появляется сообщение о успешном сохранении
    5 Нажать на кнопку загрузки фильтра на панели инструментов Открывается список ранее сохранённых фильтров
    6 Выбрать сохранённый фильтр (например, "Фильтр по банкам") и нажать "Ок" Таблица на главной форме обновляется согласно выбранному фильтру
    7 Открыть окно настройки сохранённых фильтров Открывается форма со списком сохранённых фильтров c доступными опциями редактирования и удаления
    8 Перейти на форму настройки сохранённых фильтров Открывается форма со списком сохранённых фильтров
    9 Выбрать существующий фильтр для удаления Фильтр выделен в списке
    10 Нажать на кнопку "Удалить" Появляется диалоговое окно для подтверждения удаления
    11 Подтвердить удаление Диалоговое окно закрыта, фильтр удален из списка

    13.1.3.2 Негативный сценарий - Отмена удаления фильтра¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Тестировщик видит главную страницу сервиса миграции объектов
    2 Перейти на форму настройки сохранённых фильтров Открывается форма со списком сохранённых фильтров
    3 Выбрать существующий фильтр для удаления Фильтр выделен в списке
    4 Нажать на кнопку "Удалить" Появляется диалоговое окно для подтверждения удаления
    5 Отказаться от удаления Диалоговое окно закрыто, операция удаления отменена

    13.1.4 USSDTODB-ACUS04¶

    13.1.4.1 Основной сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Пользователь видит главную страницу сервиса миграции объектов
    2 Перейти на форму "Проекты" Открыта форма "Проекты"
    3 Нажать на кнопку обновления списка проектов Список проектов обновляется, отображаются актуальные данные

    6 Выбрать модуль из списка В правой части отображаются закреплённые пользователи: Ответственный за модуль, Разработчик, Тестировщик

    13.1.5 USSDTODB-ACUS05¶

    13.1.5.1 Основной сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Пользователь видит главную страницу сервиса миграции объектов
    2 Выбрать объект БД в таблице Объект БД выделен
    3 Нажать на кнопку "Создать задачу" на панели инструментов Открывается форма создания задачи
    10 Выбрать объект БД с привязкой к проекту и модулю Объект БД выделен
    11 Нажать на кнопку "Создать задачу" Открывается форма с предварительно заполненными полями: проект, модуль (при наличии), объект, тип объекта
    12 Проверить, что поля проекта, модуля, объекта и типа объекта не редактируемы Поля заблокированы для редактирования
    13 Проверить наличие поля "Комментарий к задаче" Поле присутствует и редактируемо
    19 Заполнить поле "Исполнитель задачи" из списка Поле заполнено
    20 Нажать на кнопку подтверждения Задача создана, появляется сообщение с номером задачи

    13.1.5.2 Негативный сценарий¶

    Шаг Описание шага Результат

    2 Выбрать объект БД без привязки к проекту или модулю Объект БД выделен
    3 Нажать на кнопку "Создать задачу" Появляется сообщение о невозможности создания задачи

    13.1.5.3 Негативный сценарий¶

    Шаг Описание шага Результат
    1 Пройти авторизацию под пользователем с ролью "Руководитель" Пользователь видит главную страницу сервиса миграции объектов
    2 Выбрать объект БД с привязкой Объект БД выделен

    4 В поле "Комментарий к задаче" добавить текст, превышающий 10 тыс символов Поле заполнено
    5 Нажать на кнопку подтверждения Появляется сообщение об ошибке
